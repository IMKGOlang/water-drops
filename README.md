# 💧 Поверхность воды — интерактивная симуляция жидкости

Приложение с графическим интерфейсом: окно имитирует поверхность воды (вид сверху)
с физикой на основе уравнений Навье–Стокса. Вся симуляция считается в реальном
времени на видеокарте (WebGL, фрагментные шейдеры GLSL) — без единой библиотеки,
одним HTML-файлом.

![Симуляция](https://img.shields.io/badge/WebGL-GPU%20simulation-2f9ec7)
![Один файл](https://img.shields.io/badge/dependencies-0-success)

**▶️ Попробовать онлайн: https://imkgolang.github.io/water-drops/**

## 🚀 Запуск

Просто откройте `index.html` в любом современном браузере (двойной клик).
Ничего устанавливать не нужно: нет ни сборки, ни зависимостей, ни сервера —
всё работает локально и офлайн.

**Требования:** браузер с поддержкой WebGL (Chrome, Edge, Firefox, Safari
не старше ~2017 г.) и любая видеокарта, хотя бы встроенная.

## 🎮 Возможности

| Действие | Что происходит |
|---|---|
| **Режим «Касание»** — клик по воде | Прикосновение твёрдым телом: расходятся круги, возникает радиальное течение |
| Ведение зажатой кнопкой мыши | Тело «тянется» по воде — след из волн и вихревое течение за курсором |
| **Режим «Капля»** — клик по воде | Капля выбранного цвета и объёма падает в эту точку |
| Кнопка «Сбросить каплю» | Капля падает в случайное место |
| Цвет капли | Палитра или произвольный цвет (color picker) |
| Объём капли | 1–150 мл — масштабирует размер пятна, силу всплеска и количество красителя |
| Ползунок «Вязкость» | От почти невязкой воды до сиропа |
| Ползунок «Диффузия красителя» | Скорость расплывания красителя |
| «Очистить воду» | Полный сброс всех полей |

Краситель окрашивает воду **постепенно**, как в реальной жизни: его разносит
течением, он расплывается диффузией и разбавляется. Цвета смешиваются
субтрактивно — синяя капля + жёлтая дают зелёную воду, как настоящие чернила,
а не как свет на экране.

## 🧮 Физика

### 1. Несжимаемые уравнения Навье–Стокса (поле течения)

```
∂u/∂t + (u·∇)u = −(1/ρ)∇p + ν∇²u + F      — баланс импульса
∇·u = 0                                    — несжимаемость
```

Решаются методом расщепления операторов по схеме **«Stable Fluids»**
(Jos Stam, SIGGRAPH 1999). Шаги на каждый кадр:

- **Адвекция `(u·∇)u`** — полулагранжева схема: из каждой ячейки трассируем
  назад по потоку и берём значение там: `u(x, t+Δt) = u(x − u·Δt, t)`.
  Безусловно устойчива при любом шаге времени (`advectionShader`).
- **Проекция давления** — уравнение Пуассона `∇²p = ∇·u*` решается
  итерациями Якоби (24 итерации): `p = (p_L + p_R + p_T + p_B − div)/4`,
  затем `u = u* − ∇p` — проекция Гельмгольца–Ходжа на бездивергентное
  подпространство (`divergenceShader`, `pressureShader`, `gradientSubtractShader`).
- **Вихревая добавка (vorticity confinement)** — завихрённость `ω = ∇×u`
  возвращается в поле силой `F = ε(N×ω)`, где `N = ∇|ω| / |∇|ω||`,
  компенсируя численную диссипацию мелких вихрей (`curlShader`, `vorticityShader`).
- **Вязкость `ν∇²u`** — экспоненциальное затухание импульса `u /= (1 + k·Δt)`;
  ползунок «Вязкость» управляет `k`.

### 2. Уравнение переноса красителя (адвекция–диффузия)

```
∂C/∂t + (u·∇)C = D∇²C
```

Перенос — той же полулагранжевой адвекцией (но по более детальной сетке),
диффузия — явная схема закона Фика:
`C_new = C + D·(C_L + C_R + C_T + C_B − 4C)` (`diffusionShader`).
Распад нулевой: краситель не исчезает, а только разбавляется.

### 3. Закон Бугера–Ламберта–Бера (окрашивание воды)

```
I = I₀ · e^(−ε·C)
```

Краситель хранится как **оптическая плотность** по трём каналам, а не как цвет.
При отрисовке цвет воды умножается на `exp(−absorb·scale)` — отсюда
субтрактивное смешение и реалистичное потемнение при наложении капель
(`displayShader`).

### 4. Волновое уравнение (рябь от касаний)

```
∂²h/∂t² = c²∇²h − γ·∂h/∂t
```

`h` — высота поверхности. Дискретизация leapfrog по двум слоям времени:
`h_next = ((h_L + h_R + h_T + h_B)/2 − h_prev)·damping` (`waveShader`).
Сама высота не рисуется: из градиента `∇h` строится нормаль поверхности →
преломление (смещение выборки красителя), диффузное освещение и блик по Фонгу.

> ⚠️ Упрощение: настоящие круги на воде — дисперсионные
> гравитационно-капиллярные волны (`ω² = gk + σk³/ρ`), где скорость зависит
> от длины волны. Здесь все волны бегут с одной скоростью `c` — для глаза
> разница почти незаметна.

## ⚙️ Технические детали

- **Сетки:** скорость/давление — 192, краситель — 768, высота поверхности — 384
  (по короткой стороне окна; текстуры half-float).
- **Пайплайн кадра:** завихрённость → вихревая сила → дивергенция → давление
  (24 итерации Якоби) → вычитание градиента → адвекция скорости → адвекция
  красителя → диффузия красителя → 2 подшага волнового уравнения → отрисовка.
- **WebGL2** с откатом на WebGL1 (`OES_texture_half_float`); при отсутствии
  линейной фильтрации float-текстур — ручная билинейная интерполяция в шейдере.
- Вбросы (капля, касание) — гауссовы пятна (`splat`): краситель, радиальный
  импульс скорости и возмущение высоты. Радиус и интенсивность капли
  масштабируются как `∛объёма` (площадь пятна ~ объём^⅔).
- Падение капли анимируется DOM-элементом (Web Animations API) с квадратичным
  ускорением — всплеск происходит в момент «удара» о воду.
